# 学习计划管理器

管理 `五月每日计划.html` 中的 weeks 数组，支持查看进度、和用户交流调整计划、生成后续计划。

## 关键文件

| 文件 | 作用 |
|------|------|
| `五月每日计划.html` | weeks 数组（约815-998行），定义所有任务 |
| `checklist_state.json` | 任务完成状态、番茄钟记录、日记、每日总结 |
| `focus_state.json` | 专注监控数据（学习/摸鱼时长） |
| `focus_config.json` | 专注监控黑白名单 |

## 工作流程

### 第一步：评估当前状态

1. 读 `五月每日计划.html` 中 `const weeks = [...]` 数组，了解当前计划结构
2. 读 `checklist_state.json`，统计：
   - 已完成任务数 / 总任务数
   - 最近几天的完成率趋势
   - 番茄钟时长分布
   - 日记中用户自述的状态（疲惫、分心、进度等）
3. 读 `focus_state.json`，了解实际学习时长和摸鱼情况

### 第二步：和用户交流

根据评估结果，向用户提问。参考以下维度：

- **实际进度 vs 计划进度**：哪些章节实际学完了（可能没打卡），哪些卡住了
- **学习方式偏好**：用什么资料（方浩/张宇/李永乐等）、看视频还是直接做题、背单词用什么 App
- **时间变化**：每天实际可用时间、是否有新的干扰因素（实习/课业/健康）
- **科目优先级**：哪个科目需要加速或减速
- **任务感受**：任务量是否合适、任务描述是否清晰可执行

### 第三步：设计修改方案

修改计划时遵循以下原则：

- **过去日期不变** — 保证 `checklist_state.json` 中已完成 key 不失效
- **任务格式**：`["category", "subtag", "description"]`
  - category: `math` / `eng` / `pro` / `review`
  - subtag: 简短标签如 `数` `英` `专` `复盘`
- **周边界**：周一到周日，每 7 天一周
- **任务具体化**：包含资料名称、章节号、题目数量、预估时间
- **缓冲日**：每周日可设为缓冲/补漏日

### 第四步：应用修改

1. 用 Edit 工具修改 `五月每日计划.html` 中的 weeks 数组
2. 验证：日期序列连续、周边界在周日-周一之间、无重复或遗漏
3. 刷新 `http://127.0.0.1:8765/` 确认

## 生成后续计划

当用户要求生成当前计划结束后的新计划时：

1. 从最后一个周的结束日期往后延伸
2. 保持周结构（周一到周日）
3. 根据用户当前进度和偏好确定内容
4. 延续用户的学习资料和方法（如方浩讲义、扇贝单词、求臻机械考研等）
5. 每月末尾安排月度复盘日

## 任务描述规范

```
数学：  "方浩 [科目] 第x讲 [知识点] — 讲义+做题本（约xx min）"
英语词汇："扇贝单词 — 复习300 + 新学100（约60min）"
英语练习："六级[阅读/听力/翻译/作文]（约xx min）"
专业课： "求臻机械考研 — [章节]（约xx min）"
复盘：  "周复盘/月末复盘 xx min — [具体内容]"
```

## 注意事项

- 凌晨4点分界规则：前端 JS 将 0:00-3:59 归于前一天
- 修改 weeks 数组时只改任务内容，不改 week/day 索引结构
- 任务数变化后，旧 state key 会成为孤儿 key（无害）
- `checklist_state.json` 中的 key 格式：`{weekIndex}_{dayIndex}_{taskIndex}`
